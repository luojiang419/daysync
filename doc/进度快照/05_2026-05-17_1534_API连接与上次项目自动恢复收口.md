## 本次问题定位

1. 本地 API 实际是健康的：
   - `http://127.0.0.1:17831/api/health` 可正常返回
2. 真正导致浏览器/Vite 模式误报“未连接到本地 API”的原因是：
   - FastAPI 没有配置 CORS
   - 浏览器被跨域策略直接拦截，前端只能统一显示连接失败
3. “项目根目录和本地 SQLite 固定下来”的缺口是：
   - 之前虽然项目本身是本地目录真相源
   - 但桌面端没有记住并自动恢复“上次打开的项目根目录”

## 本次完成内容

1. 后端补 CORS 支持：
   - 允许本地开发和桌面相关 origin：
     - `http://127.0.0.1:1420`
     - `http://localhost:1420`
     - `http://127.0.0.1:4173`
     - `http://localhost:4173`
     - `tauri://localhost`
     - `http://tauri.localhost`
     - `https://tauri.localhost`
2. 前端补“上次项目自动恢复”：
   - 新增 `project-persistence.ts`
   - 创建项目后保存 `last_project_root`
   - 打开项目后保存 `last_project_root`
   - App 启动且 API 健康后，自动尝试重新打开上次项目
   - 如果上次项目目录失效，则自动清理本地记录
3. 项目首页收口：
   - 新建/打开成功提示明确说明“后续会自动恢复这个目录”
   - 打开已有项目卡片文案改为“并记住这个目录”
