## 本次问题定位

1. 之前桌面端只有在少数入口会显式执行：
   - `ensureDevApi()`
   - `waitForApiReady()`
2. 其他页面的业务请求如果正好撞上：
   - 本地 API 尚未启动完成
   - 本地 API 被替换或短暂掉线
   - 端口可用但还没到可服务状态
   就会直接抛出 `API_UNREACHABLE`
3. 根因不在某一个页面，而是在前端 `api/client.ts` 缺少统一的“桌面版自动恢复本地 API”机制

## 本次完成内容

1. 前端 API 客户端统一加自动恢复：
   - `request()` 在桌面版网络失败时，会自动：
     - 调用 `ensureDevApi()`
     - 等待 `waitForApiReady()`
     - 再自动重试原请求一次
2. 新增统一健康保障入口：
   - `ensureLocalApiReady()`
   - 用于桌面版在启动和项目操作前统一确保本地 API 已就绪
3. 应用启动链收口：
   - `App.tsx` 启动时改为走 `ensureLocalApiReady()`
   - 不再依赖分散的“手动循环 + 各自重试”
4. 项目页收口：
   - `ProjectHomePage.tsx` 的 `ensureApiConnection()` 改为统一使用 `ensureLocalApiReady()`
   - 创建项目、打开项目前都会自动保证本地 API 已经拉起并可用
5. 测试补齐：
   - 新增桌面版 API 请求失败后自动拉起并重试成功的单测
   - 新增 `ensureLocalApiReady()` 自动等待 API 就绪的单测
   - 更新 app bootstrap / project page 测试以匹配统一入口
6. 新便携包已重新编译：
   - 生成新的时间戳版本，方便直接实机测试这次修复
