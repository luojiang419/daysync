## 本次问题定位

1. 便携版 `DaySync.exe` 在当前机器上实测可把 API 拉起，健康接口约 2.2 秒可返回。
2. 当前问题更像是：
   - 用户在 API 还没完全就绪时就点击了“创建项目”/“打开项目”
   - 前端创建/打开动作没有先等待 API 真正 ready
3. 此外：
   - 网络失败时 `fetch` 抛出的不是 `ApiError`
   - UI 只能落回“创建项目失败。”这种泛化提示
4. “拍摄日期”确实不该表现成结构化日期输入，而应只是可选备注字段。

## 本次完成内容

1. 前端 API 客户端增强：
   - `fetch` 网络失败统一包装为 `ApiError("API_UNREACHABLE")`
   - 错误文案明确提示：
     - `未连接到本地 API，请稍后重试；如果是桌面版，请等待本地运行时完成启动。`
   - 新增 `waitForApiReady()`，用于重试等待本地 API
2. 项目首页增强：
   - 新建/打开项目前先做 API 连接确认：
     - 先尝试 `checkHealth()`
     - 不通则尝试 `ensureDevApi()`
     - 然后最多等待 12 次、每次 500ms
   - API ready 后再真正执行创建/打开项目
3. “拍摄日期”改为可选辅助备注：
   - 前端字段名从 `shootingDate` 改为 `shootingNote`
   - UI 标签改为 `辅助备注（可选）`
   - 输入控件从 `type="date"` 改为普通文本输入
   - 后端仍沿用 `shooting_date` 字段存储，但只作为可选文本备注传入
4. 补测试：
   - API 网络失败包装
   - `waitForApiReady` 重试
   - 项目首页备注字段显示
